1、SQL优化之查询是否存在数据

  • 常见写法

    SELECT count(*) FROM table WHERE a = 1 AND b = 2

  • 推荐写法

    SELECT id_ FROM table WHERE a = 1 AND b = 2 LIMIT 1

2、MySQL 清屏

  • Linux 通用： CTRL + L
  • 命令： system clear

3、MySQL 中的索引类型

MySQL索引有哪几种类型？

  • 主键索引：PRIMARY KEY
  • 普通索引：INDEX
  • 唯一索引：UNIQUE INDEX
  • 全文索引：FULLTEXT INDEX

前三种用的比较多，一般来说每张表都会有主键，那么主键索引一定会存在

普通索引和唯一索引一般是用来优化 SQL 的查询速度

全文索引基本不会使用，有其它的替代方案

  • 主键索引和唯一索引区别
    • 主键索引不能为 null
    • 唯一索引可以为 null

4、联合索引最左前缀匹配原则

什么是最左前缀匹配原则？

  • MySQL 建立联合索引会遵守最左前缀匹配原则，即最左优先，在检索数据时从联合索引的最左边开始匹配。
